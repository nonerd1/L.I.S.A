Control Path

The control system follows a multi-layered approach, with arm extension controlled via smartphone over Bluetooth, while flight dynamics are handled by the Pixhawk controller. This separation ensures reliable flight performance even during arm adjustment.

Communication Protocols

We implemented standard communication protocols (UART, I2C, SPI) for component interfacing, with a custom messaging protocol over Bluetooth to enable precise arm control. The Pixhawk uses the MAVLink protocol for telemetry and status monitoring.

Arm Extension Control

1// Control arm extension based on input percentage (0-100%)
2void extend_arms(uint8_t percentage) {
3  // Limit input range
4  if (percentage > 100) percentage = 100;
5  
6  // Calculate PWM duty cycle (1000-2000 Âµs)
7  uint16_t pulse_width = 1000 + (percentage * 10);
8  
9  // Apply PWM to actuator channels
10  for (int i = 0; i < NUM_ARMS; i++) {
11    // Set direction pin based on extension
12    GPIO_WriteBit(ACTUATOR_DIR_PORT, ACTUATOR_DIR_PIN[i], percentage > last_percentage[i] ? Bit_SET : Bit_RESET);
13    
14    // Set PWM value for the actuator
15    TIM_SetCompare1(ACTUATOR_TIM[i], pulse_width);
16  }
17  
18  // Store for next comparison
19  last_percentage = percentage;
20}

PID Control Settings

1// PID controller configuration for flight stabilization
2void configure_pid_controllers() {
3  // Roll PID
4  pid_roll.kp = 0.65f;
5  pid_roll.ki = 0.035f;
6  pid_roll.kd = 0.023f;
7  pid_roll.imax = 50.0f;  // Integral windup guard
8  
9  // Pitch PID
10  pid_pitch.kp = 0.65f;
11  pid_pitch.ki = 0.035f;
12  pid_pitch.kd = 0.023f;
13  pid_pitch.imax = 50.0f;
14  
15  // Yaw PID
16  pid_yaw.kp = 0.85f;
17  pid_yaw.ki = 0.045f;
18  pid_yaw.kd = 0f;
19  pid_yaw.imax = 50.0f;
20  
21  // Different PID settings for extended arms mode
22  if (arms_extended) {
23    // When arms are extended, we need different tuning
24    pid_roll.kp *= 0.85f;  // Reduce gain as moment of inertia increases
25    pid_pitch.kp *= 0.85f;
26    
27    // Adjust derivative gain to compensate for slower response
28    pid_roll.kd *= 1.2f;
29    pid_pitch.kd *= 1.2f;
30  }
31}
